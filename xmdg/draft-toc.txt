   Proposed format
     - Major Parts
           - each part has general intro/summary about it's aim
     - Each Part made up of Chapters
           - each chapter has maintainer
           - each chapter has general statement summarizing what the chapter aims to provide
     - Each chapter made up of sections

General aim and setup:
Aims:
    1. The document should describe the way to go if the reader wants to write an extension for/to Xaraya
    2. It should be clear how Xaraya Core works, what the place of the extension will be and what the difference with templating current modules is
    3. The example module is the guide
    4.
Setup:
    1. Intro to Xaraya, general operations
    2. Decision where to hack the thing/ what to hack (templates, add dyn data to modules, write own module)
    3. Module setup: what happens where
    4. Stappenplan to build a module
    5. All little things you need to know, refer to
        a. Checklist
