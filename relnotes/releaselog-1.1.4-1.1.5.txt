Release notes for Xaraya Core release 1.1.5

Release 1.1.5 (only relevant changes from 1.1.4 onwards included here)

Possible incompatible changes, please check the following:
    * Installing: see the file INSTALL.txt
    * Upgrading : see the file UPGRADE.txt

Major changes:
    * Fix for undefined $tplData error under php 5.2.6+
    * Multiple authkeys: allows a session to have more than one active authkey,
      useful for AJAX
    * ROLES: Return a consistent datatype for the getallactive user API function

General fixes and improvements:
    * Use a single timestamp for the start and the end of the session
    * Add dropdownlist support to blocks module
    * INSTALLER: Message if duplicate modules found
    * New flag for xarVarFetch(): XARVAR_VAL_REQ
    * MODULES:  More debug info in executeinitfunction.php
    * Change default theme moduservar to store theme directory instead of name
    * Remove the ignore_branch function from xaraya.rc, we use suspending now
    * Minor check for the rare cases when the modVar roles.usertimezone is empty
    * ROLES: Minor template issue in user-user_menu_form.xd
    * STYLES: No margins for nested form-containers
    * Don't translate install time constants
    * Changed soap_server class to nusoap_server to work with soapserver module 
        version 0.1.1 and nusoap library 0.7.3
    * Installing: Use invalid emails.  See why at www.donotreply.com
    * Improve the (database settings) screens in the installer:
        SQLite is now the default database if aviable.
        Suggesting a true invalid email adress for admin with a non existing TLD
    * Print theme: Fixed undefinied variable 'date' and titles as links
    * Correct Dynamic Checkbox Property so it works again with normal 
        checked="checked"
    * dynamicdata showform: add an error class to the wrapper of a dynamic data 
        property if that property is in error
    * Add current charset information to Page Cache replies.
    * Fix xarServerGetCurrentURL(): last part of path contained parameters
    * xarModURL(): Allow empty Short URLs '' -> http://www.myhost.com/
    * Fix page caching for Mozilla based browsers
    * DYNAMIC DATA: Add parameter 'fieldprefix' to DD getitem.
    * MODULES: Do not read the newer monotone directory when creating the 
        module listing
    * MODULES: Add some xarML calls
    * DYNAMIC DATA: Subform property to use tplmodule parameter
    * DYNAMIC DATA: tplmodule attribute in DD showdisplay.php
    * Dynamic_FileUpload_Property.php: pass the existing value into the validate
        function if no new value is present
    * Dynamic_FileUpload_Property.php: properly detect uploads module hook when
        the displayed object does not beling to the main module
    * Dynamic_URLIcon_Property.php: allow blank URIs to be saved, and pass
        invalid URI's back to the error handler
    * BLOCKLAYOUT: loop BL tag: explicitly create an empty object for both the
        named loop variable and its '$loop' alias
    * DYNAMIC DATA: Move commonly-repeated code for properties checkInput()
        method to the core Dynamic_Property object
    * When validating an object submitted in a form, place an error message on
        missing mandatory properties
    * MLS: xarML.php: bindDomain() method gets default value on second parameter
    * BLOCKLAYOUT: blnodes->loop: be explicit when creating the loop object
    * Pass 'style' attribute down the line in base/fileupload property
    * Update textarea property with class option
    * Ability to pass in itemid to user api get call so it can be used for
        standard hook functions
