This file contains some standard headers which we use in our code. You should 
take the headers as a bare minimum, extend at will


Notes on phpdoc package classification:
---------------------------------------
packages/subpackages: 
- variables
- templates
- server
- blocklayout
- legacy
- blocks
- config
- core
- database
  - databasetypenames
- exceptions
- logging
- multilanguage
- security
- sessions
- themes
  - themenames
- user
- modules
  - modulenames
( if a module doesn't fit a package mentioned it comes in the modules package)

Examples:
=========
privileges module:
------------------
@package security
@subpackage privileges

hitcount module:
----------------
@package modules
@subpackage hitcount
 
PHPdoc header for files:
========================

/**
 * Short description of purpose of file
 *
 * @package unassigned
 * @copyright (C) 2002 by the Xaraya Development Team.
 * @license GPL {@link http://www.gnu.org/licenses/gpl.html}
 * @link http://www.xaraya.com
 * 
 * @subpackage module name
 * @link  link to information for the subpackage
 * @author author name <author@email> (this tag means responsible person)
*/				

Make sure that the fileheader is immediately followed by another docblock, 
otherwise you'll get errors and the docblock is interpreted as the documentation
for the first php code encountered in the file.

PHPdoc header for functions:
============================

/**
 * Short description of the function
 *
 * A somewhat longer description of the function which may be 
 * multiple lines, can contain examples.
 *
 * @author  Author Name <author@email>
 * @deprec  date since deprecated <insert this if function is deprecated>
 * @access  public / private / protected
 * @param   param1 type Description of parameter 1
 * @param   param2 type Description of parameter 2
 * @return  type to return description
 * @throws  list of exception identifiers which can be thrown
 * @todo    <devname> <#> list of things which must be done to comply to relevant RFC or fix a problem
*/

  @todo should contain the developers name who is asking for the todo.. the text should be containted in the 
  @todo tag... the # attribute is optional.. but makes it easier to search where the todo is in the file
  mark the area in the code where the todo is contained by using // TODO: X where X = #



PHPdoc header for classes
=========================
/**
 * Short class description
 *
 * A somewhat longer description of the class which
 * may be on multiple lines and can contain examples
 *
 * @package unassigned - replace with packagename
 * @author Author Name <author@email>
 * @deprec date since deprecated <insert if class is deprecated>
 * @todo  todo-item, specify each with new tag
 */
